# Code Of Conduct

## Our Standard

This project expects respectful, direct, and technically honest collaboration.

Examples of behavior that help:

- giving actionable technical feedback
- assuming good intent while challenging weak reasoning
- documenting risks and tradeoffs clearly
- respecting boundaries around secrets, private data, and user environments

Examples of unacceptable behavior:

- harassment or personal attacks
- abusive, discriminatory, or sexualized language
- doxxing or sharing private information without permission
- bad-faith derailment, intimidation, or repeated hostile behavior

## Enforcement

Project maintainers may remove comments, issues, pull requests, or participants that violate this code of conduct.

## Reporting

For conduct issues, open a maintainer-facing issue only if it can be done safely without escalating the situation publicly. For security-sensitive material, use the process in [SECURITY.md](SECURITY.md).
